LibraryTag

This program was created to get MBIDs from the MusicBrainz service and store them in MP3 or FLAC files in a library. A music player can then retrieve further MusicBrainz data while playing the music files.

Prerequisites

A Java 8+ JRE or JDK.

On Windows, download the x64 installer or zipped archive from the official Oracle or OpenJDK website, run the setup, and configure the JAVA_HOME environment variable and add %JAVA_HOME%\bin to the Path. If all goes well, you can run java -version from any command prompt. The latest JDK-25 does not work, see here.

Installation and Removal

If you received the program as a ZIP archive, unzip it into an empty folder. Drag the JAR file onto your desktop while pressing the Alt key to create a shortcut there. Make sure you do not move or copy the file. Open the properties of the shortcut. On the Shortcut tab, you can choose the icon from the application directory.

The application itself does not use the Windows registry. To remove it, simply delete the directory including all files and the desktop shortcut.

After installation, take a look at LibraryTag.prp to edit properties.

Usage

This is a pre-release, so not all MusicBrainz functions are supported yet. The application should run on any platform where a Java 8+ runtime is available. This includes Linux, macOS, Unix, and Windows.

How to use:

  1. Start the program.
  2. Copy a valid Artist-MBID from the MusicBrainz website using Ctrl+C and paste it into the input field using Ctrl+V.
  3. Press the Go button or Enter.
  4. Requested data will be printed in the output area. Select a Release-MBID in the output area, copy it with Ctrl+C, paste it into the input field, and press Enter.
  5. The releases will be listed at the end of the output area. Repeat for a Recording-MBID.

If the settings in the properties file allow it, your library will be scanned for corresponding files.

Finding Media Library Files

For finding available media files, LibraryTag has a directory specification for how a consistent library should be organized. The structure used is:


Drive:\FileType\Artist*\*Album\*Tracks

Examples

m:\FLAC\The Beatles\1964-01 Beatles For Sale\08 - Eight Days A Week.flac
m:\MP3\Yes\1973-LL Yessongs\06 - Mood For A Day.flac
/mnt/music/OGG/King Crimson/1981-08 Discipline/Frame by Frame.ogg

If your library does not meet this specification, you have two options:

  1. Create symbolic links using the Windows tool mklink on the specified drive.
  2. Re-organize your library.
